# 1. 문제정의서 (프론트엔드)

## 1. 배경
할 일 앱은 기능이 많아질수록 입력은 늘고 실행은 줄어드는 경향이 있다.  
`5늘할일`은 하루 목표를 최대 5개로 제한해 사용자가 "오늘 해야 할 일"에 집중하도록 돕는 모바일 웹 서비스다.

## 2. 문제 정의
- 사용자는 할 일을 무한히 추가하며 실제 실행률이 낮아진다.
- 미래 계획과 오늘 할 일이 한 화면에 섞여 우선순위가 흐려진다.
- 미완료 항목이 누적되면 심리적 부담이 커진다.
- 과거 기록/보관함은 개인 데이터 성격이 있어 인증 없는 접근이 위험하다.

## 3. 프론트엔드 관점 핵심 과제
- 모바일 기준으로 빠르게 읽고 조작할 수 있는 화면 설계
- "최대 5개 제한"을 명확히 이해시키는 입력 경험
- 미완료 항목의 삭제/보관 분기를 실수 없이 처리하는 인터랙션
- 로그인 상태에 따른 화면 접근 제어(가드/리다이렉트)

## 4. 목표
- 사용자가 첫 진입 후 짧은 시간 안에 오늘 할 일을 등록/완료할 수 있다.
- 할 일 관리의 범위를 "오늘"로 제한해 실행 집중도를 높인다.
- 보관함/과거 리스트는 인증 사용자에게만 안정적으로 노출한다.

## 5. 비목표
- 미래 계획 캘린더 기능
- 복잡한 협업 기능(공유, 멤버 역할, 승인 플로우)
- 데스크톱 중심 고밀도 화면 최적화
