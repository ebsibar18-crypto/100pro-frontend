# 7. 권장개발순서

## 개발 원칙
- 핵심 가치(오늘 집중, 최대 5개 제한)를 먼저 구현한다.
- 인증은 보호가 필요한 화면(보관함/과거 조회)에 우선 적용한다.
- 공통 레이아웃/컴포넌트를 먼저 만들어 화면별 재작업을 줄인다.

## 0) 기술 스택 및 아키텍처 결정
- 개발 언어: TypeScript
- 프레임워크: React + Vite
- 라우팅: React Router
- 스타일링: Tailwind CSS(또는 CSS Modules 중 1개 확정)
- 상태 관리:
  - 서버 상태: TanStack Query
  - 클라이언트 상태: Zustand
- 폼/검증: React Hook Form + Zod
- 테스트: Vitest + React Testing Library + Playwright(E2E)
- 코드 품질: ESLint + Prettier + Husky + lint-staged
- 인증 방식: 일반 이메일 로그인 + Kakao SSO
- 완료 기준:
  - 스택/라이브러리 목록 확정
  - 인증/토큰 저장 전략 확정(HttpOnly Cookie 권장)
  - 환경변수 규칙(dev/stage/prod) 문서화 완료

## 1) 정보구조(IA) 및 라우팅 확정
- 화면: Splash, Home(오늘 할 일), Archive(보관함), Login
- 규칙: 비로그인 상태에서 Archive 접근 시 Login으로 리다이렉트
- 완료 기준:
  - 화면/경로 목록 확정
  - 접근 제어 규칙 문서화 완료

## 2) 모바일 레이아웃 뼈대 + 디자인 시스템
- 모바일 기준 컨테이너, 카드, 체크박스, 하단 내비 컴포넌트 구축
- 타이포 가이드(온글잎 폰트 + fallback) 적용 기준 정리
- 완료 기준:
  - 공통 컴포넌트 재사용 가능
  - 주요 화면에서 레이아웃 깨짐 없음

## 3) Home(오늘 할 일) 핵심 기능 구현
- 오늘 할 일 생성/수정/삭제/완료 토글
- 최대 5개 제한 및 안내 문구
- 미래 날짜 작성 경로 제거
- 완료 기준:
  - 5개 초과 생성 불가
  - 상태(진행 전/중/완료) 표시 동작

## 4) 미완료 항목 정리 플로우 구현
- 삭제/보관 선택 모달(또는 바텀시트) 제공
- 삭제 시 즉시 제거, 보관 시 보관함 이동
- 완료 기준:
  - 두 옵션 모두 정상 동작
  - 사용자 확인 단계(실수 방지) 반영

## 5) 보관함(Archive) 화면 구현
- 보관 목록, 완료 구분 영역, 빈 상태 UI 구현
- Home에서 보관된 항목이 즉시 반영되도록 연결
- 완료 기준:
  - 보관 항목 조회 가능
  - 빈 상태/완료 상태 표현 정상

## 6) 인증 구현 (일반 이메일 + Kakao SSO)
- 로그인 화면 구현
- Archive/과거 조회 보호 라우팅 적용
- 로그인 성공 후 원래 목적 화면 복귀 처리
- 완료 기준:
  - 비로그인 보호 동작 확인
  - 로그인 후 복귀 동선 확인

## 7) 스플래시/브랜딩 마무리
- 로고/카피/전환 타이밍 적용
- 초기 진입 경험(로딩 체감) 정리
- 완료 기준:
  - 진입 흐름 자연스러움
  - 브랜딩 일관성 확보

## 8) QA 및 안정화
- 접근성(터치 영역, 대비), 오류 상태, 빈 상태 점검
- 핵심 시나리오 E2E 검증
  - 앱 진입 -> 오늘 할 일 작성(최대 5개) -> 미완료 보관
  - 보관함 접근(비로그인) -> 로그인 -> 보관함 복귀
- 완료 기준:
  - 핵심 플로우 결함 없음
  - 문서(PRD/SED/결정로그/백로그)와 구현 일치

## 간단 일정 예시
- Week 1: 1~3단계
- Week 2: 4~6단계
- Week 3: 7~8단계 + 안정화
