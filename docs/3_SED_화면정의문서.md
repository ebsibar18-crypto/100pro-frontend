# 3. SED 화면정의문서 (Screen Experience Definition)

## 문서 목적
이 문서는 모바일 웹 기준 화면 구조, 상태, 전환 규칙을 정의한다.

## 1. 화면 목록

### 1) 스플래시/로딩
- 요소: 로고, 태그라인
- 폰트: 온글잎 박다현, 온글잎 김콩해(브랜드 영역 우선 적용)
- 전환: 앱 초기 로딩 완료 시 오늘 할 일 화면으로 이동
- 관련 백로그: PRO-F-2, PRO-F-10

### 2) 오늘 할 일 화면 (핵심)
- 구조:
  - 상단 타이틀
  - 상태 그룹(진행 전/진행 중/진행 완료)
  - 항목 카드 + 체크박스
  - 하단 네비게이션
- 규칙:
  - 할 일은 최대 5개까지 생성 가능
  - 미래 날짜 작성 경로 없음
- 관련 백로그: PRO-F-1, PRO-F-3, PRO-F-4, PRO-F-5

### 3) 미완료 정리 인터랙션
- 트리거: 사용자가 정리 액션 실행 시
- 옵션:
  - 삭제: 즉시 제거
  - 보관: 보관함으로 이동
- UI 형태: 모달 또는 바텀시트
- 관련 백로그: PRO-F-6

### 4) 보관함 화면
- 구조:
  - 보관 항목 목록
  - 완료 처리된 보관 항목 구분 영역
- 접근 제어:
  - 비로그인 시 로그인 화면으로 이동
- 관련 백로그: PRO-F-7, PRO-F-8

### 5) 로그인 화면
- 소셜 로그인(카카오 SSO) + 일반 이메일 로그인
- 성공 시 직전 접근 화면으로 복귀
- 관련 백로그: PRO-F-9

## 2. 상태 정의
- `Unauthenticated`: 보관함 접근 불가
- `Authenticated`: 보관함/과거 리스트 접근 가능
- `TodoCountReached`: 오늘 할 일 5개 도달 상태
- `ArchiveMovePending`: 미완료 항목 정리 대기 상태

## 3. 전환 규칙
- 스플래시 -> 오늘 할 일
- 오늘 할 일 -> 보관함(비로그인): 로그인으로 리다이렉트
- 로그인 성공 -> 이전 목적 화면(보관함) 복귀
- 오늘 할 일에서 미완료 정리 선택:
  - 삭제 -> 목록 제거
  - 보관 -> 보관함 항목 추가

## 4. 접근성/사용성 기준
- 터치 영역은 모바일 엄지 조작을 고려한 크기로 유지
- 완료/미완료/비활성 상태는 색상 + 아이콘으로 동시에 전달
- 오류/빈 상태 화면을 명확히 제공

## 5. 관련 백로그
- PRO-F-1, PRO-F-2, PRO-F-5, PRO-F-6, PRO-F-7, PRO-F-8, PRO-F-9, PRO-F-10, PRO-F-11, PRO-F-12
