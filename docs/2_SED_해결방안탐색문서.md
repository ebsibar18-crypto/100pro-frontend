# 2. SED 해결방안탐색문서 (Solution Exploration)

## 문서 목적
이 문서는 `5늘할일`의 프론트엔드 문제를 해결하기 위한 대안들을 비교하고, 채택할 방향을 기록한다.

## 1. 해결해야 할 질문
1) 왜 "최대 5개 제한"이 사용자에게 납득 가능한가?  
2) 미완료 항목을 어떤 방식으로 정리해야 부담이 가장 낮은가?  
3) 보관함/과거 리스트 접근을 어디서 인증할 것인가?

## 2. 이슈별 대안 탐색

### A. 오늘 할 일 개수 제한 (PRO-F-3)
**대안 A1: 입력 시점 제한**
- 5개 도달 시 입력창/추가 버튼 비활성
- 장점: 즉시 이해 가능, 구현 단순
- 단점: 왜 제한되는지 설명 없으면 답답함

**대안 A2: 저장 시점 제한**
- 입력은 가능, 저장 시 경고 후 차단
- 장점: 사용 흐름을 덜 끊음
- 단점: 늦은 피드백으로 좌절감 발생 가능

**채택: A1 + 안내 문구 병행**
- 이유: 모바일에서는 즉시 피드백이 오류 비용을 줄임
- 반영 백로그: PRO-F-3

### B. 미래 계획 제한 (PRO-F-4)
**대안 B1: 날짜 선택 UI 자체 제거**
- 오늘 화면만 제공
- 장점: 정책 일관성, 화면 단순화
- 단점: 일부 사용자가 기능 부족으로 느낄 수 있음

**대안 B2: 날짜 선택은 보이되 미래는 비활성**
- 장점: 정책을 시각적으로 전달
- 단점: UI 복잡도 증가

**채택: B1**
- 이유: 제품 철학(오늘 집중)과 가장 일치
- 반영 백로그: PRO-F-4

### C. 미완료 항목 처리 (PRO-F-6, PRO-F-7)
**대안 C1: 미완료 자동 보관**
- 장점: 데이터 손실 없음
- 단점: 사용 의도와 달라도 보관 누적

**대안 C2: 삭제/보관 선택 모달 제공**
- 장점: 사용자 통제권 확보, 의사결정 명확
- 단점: 클릭 수 증가

**채택: C2**
- 이유: 서비스 핵심(오늘 집중)과 기록 보존의 균형
- 반영 백로그: PRO-F-6, PRO-F-7

### D. 인증 시점 (PRO-F-8, PRO-F-9)
**대안 D1: 앱 진입 즉시 로그인 필수**
- 장점: 보호 강함
- 단점: 초기 이탈 증가

**대안 D2: 핵심 사용(오늘 할 일)은 비로그인 허용, 보관함/과거 조회 시 로그인 요구**
- 장점: 온보딩 장벽 낮음, 보호 구간 명확
- 단점: 중간 전환 플로우 설계 필요

**채택: D2**
- 이유: 첫 사용성 확보 + 개인 기록 보호 균형
- 반영 백로그: PRO-F-8, PRO-F-9

## 3. 채택 원칙
- 오늘 집중을 해치지 않는가?
- 모바일 조작 비용이 낮은가?
- 상태 전이가 사용자에게 명확한가?
- 인증이 필요한 데이터만 보호하는가?

## 4. 결정 요약
- 최대 5개 제한: 입력 시점 차단 + 이유 안내
- 미래 계획: UI에서 제외
- 미완료 처리: 삭제/보관 선택
- 인증 정책: 보관함/과거 조회 시점 인증

## 5. 후속 검증 항목
- 5개 제한 안내 문구 A/B 테스트
- 미완료 정리 모달의 선택률/이탈률 측정
- 인증 유도 시점의 전환율 측정

## 6. 관련 백로그
