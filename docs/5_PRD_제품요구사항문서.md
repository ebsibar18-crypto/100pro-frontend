# 5. 제품요구사항문서 (PRD) - 5늘할일

## 1. 제품 개요
`5늘할일`은 하루 목표를 최대 5개로 제한해 사용자가 "오늘 할 일" 실행에 집중하도록 돕는 모바일 웹 Todo 서비스다.

## 2. 타깃 사용자
- 해야 할 일이 많아 우선순위 정리에 어려움을 겪는 사용자
- 복잡한 기능보다 빠른 입력/체크를 선호하는 사용자
- 하루 단위 루틴을 가볍게 유지하고 싶은 사용자

## 3. 핵심 가치
- 많이 계획하기보다 오늘 할 일을 끝내는 경험
- 미완료 항목을 삭제/보관으로 정리하는 부담 완화
- 기록 조회는 인증으로 보호하는 신뢰성

## 4. 기능 요구사항 (Frontend)

### FR-01 모바일 레이아웃 프레임
- 모바일 우선 레이아웃/여백/하단 탭 구조를 제공한다.
- 백로그: PRO-F-1

### FR-02 스플래시/로딩
- 앱 진입 시 로고/카피를 보여주고 메인 화면으로 전환한다.
- 백로그: PRO-F-2

### FR-03 오늘 할 일 5개 제한
- 오늘 할 일은 최대 5개까지만 생성 가능하다.
- 5개 도달 시 추가 생성 CTA를 비활성화하고 안내 문구를 노출한다.
- 백로그: PRO-F-3

### FR-04 미래 일정 제한
- 미래 날짜 기준의 입력 경로를 제공하지 않는다.
- 백로그: PRO-F-4

### FR-05 상태 분류 UI
- 진행 전/진행 중/진행 완료 상태를 구분해 표시한다.
- 백로그: PRO-F-5

### FR-06 미완료 항목 정리
- 미완료 항목에 대해 삭제/보관 선택 UI를 제공한다.
- 삭제는 즉시 제거, 보관은 보관함 이동으로 처리한다.
- 백로그: PRO-F-6

### FR-07 보관함
- 보관 항목 목록 조회 화면을 제공한다.
- 빈 상태/완료 상태를 구분해 표시한다.
- 백로그: PRO-F-7

### FR-08 접근 인증 가드
- 비로그인 사용자가 보관함/과거 리스트 접근 시 로그인 화면으로 리다이렉트한다.
- 백로그: PRO-F-8

### FR-09 로그인 경험
- 카카오 SSO/일반 이메일 로그인 UI를 제공한다.
- 로그인 성공 시 이전 목적 화면으로 복귀한다.
- 백로그: PRO-F-9

### FR-10 타이포/브랜딩
- 온글잎 박다현, 온글잎 김콩해 적용 지침을 정의한다.
- 폰트 로딩 실패 시 폴백 폰트를 사용한다.
- 백로그: PRO-F-10

### FR-11 접근성
- 터치 영역, 대비, 상태 전달 기준을 만족한다.
- 백로그: PRO-F-11

### FR-12 빈 화면/오류 처리
- 데이터 없음, 인증 실패, 네트워크 오류 상태 UI를 제공한다.
- 백로그: PRO-F-12

## 5. 비기능 요구사항
- 모바일 기기에서 주요 상호작용의 체감 지연 최소화
- 주요 상태 변경(체크/삭제/보관)은 즉시 시각 반영
- 일관된 컴포넌트 스타일 유지

## 6. 수용 기준 (Acceptance Criteria)
- [ ] 할 일 5개 초과 생성이 불가능하다. (PRO-F-3)
- [ ] 미래 일정 입력 경로가 없다. (PRO-F-4)
- [ ] 미완료 항목은 삭제/보관으로 처리된다. (PRO-F-6)
- [ ] 보관함 접근 시 비로그인은 로그인으로 이동한다. (PRO-F-8)
- [ ] 로그인 성공 시 의도한 화면으로 복귀한다. (PRO-F-9)
- [ ] 모바일 화면에서 레이아웃 깨짐 없이 동작한다. (PRO-F-1)
